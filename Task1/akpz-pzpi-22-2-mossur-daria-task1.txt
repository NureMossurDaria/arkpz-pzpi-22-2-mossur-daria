МЕТА РОБОТИ

	Метою лабораторної роботи є створення документації типу Vision & Scope та формування аркуша завдання для програмної системи донорства. Ці документи призначені для визначення основної концепції проєкту, встановлення його меж та формулювання вимог, що забезпечать його успішну реалізацію


ХІД РОБОТИ


	Розробка документації розпочалася з формування розділу бізнес-вимог. У цьому розділі було проведено детальний аналіз ситуації на ринку програмних систем для донорства крові та обґрунтовано необхідність створення нового рішення. Було досліджено конкурентні продукти та визначено їх сильні та слабкі сторони.
	При роботі над розділом концепції рішення було розроблено структуру системи з описом всіх її складових частин. Особлива увага приділялася опису взаємодії між компонентами та визначенню ключових функціональних можливостей кожного модуля.
	У процесі роботи над розділом обмежень проєкту було сформовано чіткий перелік функцій, які повинні бути реалізовані у першій версії продукту. Також було визначено технічні обмеження системи та сформовано план подальшого розвитку продукту.
	Завершальним етапом стало створення розділу бізнес-контексту, де було проаналізовано всі зацікавлені сторони проєкту та встановлено часові рамки його реалізації. Було визначено необхідні ресурси та сформовано критерії оцінки успішності проєкту.

1. БІЗНЕС-ВИМОГИ
1.1. Передумови
	У сучасному світі потреба в ефективному управлінні запасами донорської крові стає все більш критичною та нагальною. Медичні заклади різного профілю щодня стикаються з необхідністю забезпечення постійного доступу до донорської крові, що безпосередньо впливає на можливість рятування людських життів.
	Кількість спеціалізованих медичних центрів та відділень, які потребують регулярного постачання крові різних груп та резус-факторів, постійно зростає, створюючи додатковий тиск на існуючу систему управління запасами. Традиційні методи залучення донорів та координації процесів донації виявляються недостатньо ефективними в умовах, коли кожна хвилина може бути вирішальною для пацієнта.
	Особливо гострим питанням залишається забезпечення належних умов зберігання донорської крові, що вимагає постійного моніторингу температурних режимів та інших критичних параметрів середовища для підтримання її якості та безпечності.
	Додатковим викликом стає необхідність обробки та збереження величезних масивів даних, включаючи інформацію про донорів, їхній стан здоров'я, історію донацій та запити від медичних установ. Це потребує потужної інфраструктури та надійних механізмів захисту даних.
	Враховуючи всі ці фактори, впровадження сучасної інтегрованої системи управління донорською кров'ю стає не просто бажаним, а необхідним кроком для забезпечення ефективного функціонування системи охорони здоров'я та своєчасного надання медичної допомоги
1.2. Бізнес-можливості 
	Хоча програмна система управління процесом донорства та запасами крові вирішує багато проблем медичних працівників і керівництва лікарень, насправді якісних аналогів, що охоплюють усі аспекти – від залучення донорів до контролю умов зберігання крові, – доволі мало. Прикладами існуючих рішень можуть бути «Red Cross Blood Donor», «Hemasoft Blood Bank Management» та «Donor Info».
	«Red Cross Blood Donor» (посилання) орієнтується на залучення донорів завдяки мобільним сповіщенням про нагальні потреби в крові. Однак її функціональність зводиться переважно до інформування донорів, і вона не забезпечує масштабного управління складськими запасами крові чи детального аналітичного інструментарію. Також тут відсутня інтеграція з датчиками для контролю умов зберігання, що ускладнює підтримку стабільної якості крові.
	«Hemasoft Blood Bank Management» (посилання) має більший набір функцій для керування банком крові, включно з базою даних донорів та інструментами для аналізу наявних запасів. Ця система підходить для великих лікарень, надає детальні звіти та підтримує інтеграцію з лабораторними комплексами. Проте їй бракує зручного мобільного застосунку з можливістю запису на донацію безпосередньо самими донорами. Також «Hemasoft» не реалізує власних IoT-рішень для моніторингу холодильників із кров’ю.
	«Donor Info» (посилання) здебільшого орієнтується на невелику територіальну громаду чи окреме місто. Вона надає змогу швидко повідомляти донорів про необхідність у певній групі крові й вести спрощену базу. Водночас система не має розгорнутих інструментів планування, аналітики або адаптації під великі медичні мережі. Крім того, відсутні будь-які модулі з контролю умов зберігання крові.
	Якщо порівнювати наведені аналоги з даною системою, то можна впевнено стверджувати, що наша програмна розробка матиме ширші можливості. Передусім планується інтеграція з IoT-пристроями для контролю температури та вологості в холодильниках і централізоване керування запасами крові. Також система передбачає повноцінний веб-інтерфейс для медичних працівників та мобільний додаток для донорів, що дасть змогу автоматизувати запис на донацію, відстежувати історію кровоздач та своєчасно надсилати сповіщення про критичні потреби.
	Модель монетизації для програмного продукту управління донорством і запасами крові може включати ліцензії для медичних установ на встановлення та використання системи; підписку за моделлю SaaS для зручного доступу до хмарної платформи, а також продаж датчиків та обладнання з вбудованою підтримкою програмної системи. Також може бути передбачена адаптація системи для конкретних великих мереж чи регіонів із окремим тарифним планом.
	Ринок, на який орієнтується цей програмний продукт, включає державні та приватні лікарні, станції переливання крові, центри донорства та великі медичні об’єднання. З урахуванням зростаючої уваги до оптимізації витрат, необхідності підвищення безпеки зберігання крові та спрощення взаємодії з донорами, такий комплексний підхід може мати перспективи на ринку програмних рішень для охорони здоров’я.

2. КОНЦЕПЦІЯ РІШЕННЯ 
2.1. Окреслення концепції 
	Головною метою даного проєкту є створення комплексного рішення для оптимізації процесу донорства крові та управління запасами крові у медичних закладах. Проєкт спрямований на вирішення критичних проблем галузі, серед яких: неефективна комунікація між донорами та медичними закладами, складність відстеження стану запасів крові, та відсутність автоматизованої системи моніторингу умов зберігання.
	Впровадження цієї системи забезпечить повний цикл управління донорством крові через чотири взаємопов'язані компоненти:
	Веб-платформа для медичного персоналу створює єдиний центр управління донорством, що дозволяє медичним працівникам оперативно формувати запити на кров, контролювати наявні запаси та координувати донорські заходи. Це значно підвищує ефективність роботи медичного персоналу та скорочує час реагування на критичні потреби в крові.
	Мобільний застосунок революціонізує взаємодію з донорами, надаючи їм зручний інструмент для участі у донорстві. Система сповіщень про нагальні потреби, функція запису на донацію та персоналізовані рекомендації щодо підготовки створюють безперервний цикл залучення донорів.
	Серверна частина виступає мозковим центром системи, забезпечуючи інтелектуальну обробку даних та автоматизоване співставлення запитів на кров з доступними донорами. Це дозволяє оптимізувати розподіл ресурсів та швидко реагувати на критичні ситуації.
	Підсистема IoT реалізує безперервний моніторинг умов зберігання крові через мережу спеціалізованих датчиків. При відхиленні показників від норми система миттєво генерує сповіщення та синхронізує дані з центральним сервером, забезпечуючи збереження якості запасів крові
2.1. Головна функціональність 
	ГФ-1: Реєстрація та авторизація користувачів
		Створення облікових записів медичних працівників через веб-інтерфейс
		Реєстрація донорів через мобільний застосунок
	ГФ-2: Створення та управління запитами на кров
		Формування запитів із зазначенням групи крові та резус-фактора
		Автоматичний пошук та підбір донорів за вказаними критеріями
	ГФ-3: Запис на донацію
		Вибір зручного місця та часу для донації через мобільний застосунок
		Система підтверджень та нагадувань про заплановану донацію
	ГФ-4: Відстеження донацій та медичних показників
		Доступ медичних працівників до повної історії донацій та аналізів
		Особистий кабінет донора з історією та запланованими донаціями
	ГФ-5: IoT-моніторинг умов зберігання
		Збір даних про температуру та вологість у холодильниках
		Система сповіщень при відхиленні від допустимих параметрів
	ГФ-6: Система нагадувань про донації
		Автоматичні сповіщення про можливість повторної донації
		Врахування встановлених інтервалів між донаціями

3. Рамки та обмеження 
3.1. Рамки первинного випуску 
Для кращого опису функціональності системи управління донорською кров'ю у першому випуску програмного продукту, поділімо функції на частини, включаючи серверну частину, частину IoT, веб-частину та мобільний додаток.
Розпочнімо із функціональності серверної частини:
	1. Автентифікація та авторизація. Сервер має забезпечити систему автентифікації медичних працівників та донорів, надання прав доступу на основі ролей;
	2. Збереження даних. Сервер повинен забезпечити зберігання даних про донації, користувачів, історії донацій, запаси крові та їх характеристики;
	3. Локалізація. Можливість локалізації інтерфейсу для різних мов та країн;
	4. Захист даних. Всі важливі медичні дані та особиста інформація мають бути зашифровані та при необхідності дешифровані на серверній частині;
	5. Автоматичне сповіщення. Сервер повинен містити необхідну бізнес-логіку для розсилки термінових сповіщень про потребу в донорах;
	6. Інвентаризація. Сервер повинен здійснювати аналіз всіх запасів крові та знаходити зразки, термін придатності яких спливає. Система надсилає повідомлення про такі зразки відповідальним особам;
	7. Обробка запитів на донацію. Сервер повинен швидко обробляти запити на донацію та надсилати підтвердження донорам;
	8. Адміністрування. Сервер повинен враховувати наявність ролей медичного персоналу з різними рівнями доступу до даних.
Функціональність для IoT частини:
	1. Збір даних про температуру та вологість у холодильниках для зберігання крові;
	2. Моніторинг та контроль температури, з негайним сповіщенням при відхиленні від норми більше ніж на 2-3 градуси;
	3. Аналіз оптимального часу для проведення інвентаризації запасів крові;
	4. Передача даних на сервер для подальшої обробки;
	5. Аналітичний інструмент для прогнозування потреб у донорській крові.
Клієнтська частини, веб-частина:
	1. Графічний інтерфейс для авторизації медичного персоналу;
	2. Різні інтерфейси для різних категорій медичних працівників;
	3. Локалізація інтерфейсу;
	4. Управління запитами на донацію;
	5. Налаштування особистого кабінету;
	6. Доступ до інформації про стан зберігання крові.
Мобільний додаток матиме такі функції:
	1. Реєстрація та авторизація донорів;
	2. Перегляд історії власних донацій;
	3. Запис на донацію;
	4. Отримання сповіщень про терміновий пошук донорів;
	5. Доступ до особистого медичного профілю.
3.2. Рамки наступних випусків
Функції, які планується реалізувати у майбутньому:
	1. Розширена аналітика потреб у донорській крові;
	2. Інтеграція з лабораторними системами;
	3. Розширена програма мотивації донорів.

4. Бізнес-контекст 
4.3. Робоче середовище
	Для реалізації програмної системи для донорства буде використано сучасні технології, які забезпечать стабільність, масштабованість, продуктивність та інтеграцію компонентів системи. Нижче наведено перелік технологій, які будуть використовуватись при створенні кожної частини системи.
	Для створення Веб-сайту буде використано мову програмування Java Script, для оформлення веб-інтерфейсу, забезпечення сучасного дизайну та адаптивності – CSS, а також HTML для створення структури сторінок. Для розробки серверної частини програмної системи буде використано такі технології: сучасна версія мови програмування Java 21, Spring Framework, Spring Boot та Spring Web – для розробки RESTful API, що забезпечить зв’язок між компонентами системи. База даних PostgreSQL – реляційна база даних, що забезпечує високу продуктивність, використовуватиметься для зберігання даних про донорів, історії донорства, заявок та іншої інформації. Java 21, Spring Framework, Spring Boot, Spring Web – технології для створення застосунку для smart-пристроїв (IoT). Вони дозволяють інтегрувати smart-пристрої, які будуть використовуватися для управління обладнанням (холодильниками для зберігання крові). Для розробки мобільного застосунку будуть використовуватись мова програмування Kotlin, Android SDK та Jet	pack Compose.
